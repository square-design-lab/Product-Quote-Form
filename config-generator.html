Plugin Mode

Button Labels

Replaces the native "Add to Cart" button text on product pages.

Button Visibility

Keep the original Add to Cart alongside the Enquiry button on product pages.

Keep the Checkout button alongside the Quote button on the cart page.

Hide prices on product pages — ideal for "Price on Request" use cases.

Tag Filtering

Setup Guide

        
Follow these steps to install the Product Enquiry plugin on your Squarespace site.


        

          
1

          

            
Create the Lightbox Form

            
Go to your Squarespace site editor. Open the Footer section. Add a Form Block and set it to open as a Lightbox. Design the form with the fields you want (Name, Email, Phone, Message, etc.). Add one Textarea field and set its label to match the Form Field Label setting in the config (default: Cart). This field will be auto-filled with product/cart data. You may want to hide this field with CSS later.

          

        


        

          
2

          

            
Configure Form Storage

            
Make sure the form is set to send submissions to your email (Form → Storage → Email). You'll receive product details directly in the email when customers submit enquiries.

          

        


        

          
3

          

            
Configure the Plugin

            
Use the sidebar controls to configure your plugin settings. Choose your mode (Quote Cart, Single Product, or Both), set your button labels, and adjust visibility options. If you want to apply the enquiry only to specific products, enable Tag Filtering and add a tag like enquiry to those products.

          

        


        

          
4

          

            
Copy the Code

            
Go to the Code Output tab (or use the Copy Code button in the sidebar header), copy the full code snippet, and paste it into Settings → Advanced → Code Injection → Header in your Squarespace admin.

          

        


        

          
5

          

            
Tag Your Products (if using tag filtering)

            
If tag filtering is enabled, go to each product you want to add enquiry functionality to and add the tag you configured (default: enquiry). Products without the tag will keep the normal purchase flow. Make sure the tag in Squarespace exactly matches the Enquiry Tag value in your config.

          

        


        

          
6

          

            
Checkout vs. Quote

            
Important: Squarespace uses a single cart for all products. If you're using this plugin to replace checkout with quotes, we recommend hiding the Checkout button (disable "Show native Checkout" in the sidebar). When tag filtering is enabled, only tagged products will appear in the quote form — but the native checkout would still process all cart items, which can cause confusion. For stores where some products are for purchase and others for quoting, consider using separate store pages.

          

        


        

          
7

          

            
Optional: Hide the Cart Field

            
To hide the auto-filled "Cart" textarea from the popup form, add this CSS to Design → Custom CSS:

            
.lightbox-inner .field-list .form-item:has(label[for] .title:not(:empty)) { /* target the Cart field by its wrapper */ }

            
Or simply position it off-screen. The exact CSS depends on your form structure — enable Debug Mode to inspect elements.

          

        


        

          
8

          

            
Test & Go Live

            
Visit a product page and click the Enquiry button (or add products to cart and test the quote form). Verify that the form popup shows the correct product details. Submit a test enquiry and check your email. Disable Debug Mode when you're satisfied.
